# PRD：TikTok 品牌广告 MMM 指标相关性分析工具

**版本**：v0.1 (Draft)
**作者**：Iris（TikTok Brand Ads PM）
**最后更新**：2026-04-19
**目标读者**：PM 团队（自用 / 内部协作）

## 1. 背景与目标

### 1.1 业务背景

TikTok 品牌广告（Brand Ads）客户越来越多地使用 **Media Mix Modeling (MMM)** 来衡量跨渠道的真实增量贡献和 ROI。但在日常优化中，品牌广告的投放侧指标（CTR、VTR、CPM、Frequency 等）与客户 MMM 输出的 ROI 之间存在 **"体感有关、但说不清哪个最关键"** 的脱节：

- 品牌客户常问："我应该把 frequency 控制在多少？CTR 多高算健康？"——我们目前只有行业经验值，没有基于该客户自身 MMM 结果的个性化回答。
- 优化侧的 PM/AM 缺乏一个标准化工具来快速回答："这次投放中哪个指标波动最能解释 MMM ROI 的变化？"
- 跨客户、跨行业的 learning 难以沉淀——每个 case 都靠 Excel 手搓，无法对比。

### 1.2 产品目标

构建一个内部分析工具，PM 输入多周期的广告投放指标 + 对应的 MMM ROI 结果，工具自动输出：

1. **相关性排序**：哪些指标与 MMM ROI 相关性最强（含方向和置信度）。
2. **最佳区间识别**：每个关键指标在什么数值区间内 MMM ROI 表现最佳。
3. **可解释的 insight 卡片**：输出可直接用于客户沟通/内部复盘的结论。

### 1.3 非目标（Non-goals，V1 不做）

- 不做 MMM 建模本身（假设 MMM ROI 由客户或第三方已给出）。
- 不做实时投放优化建议回写到广告系统。
- 不做多客户聚合分析（V1 只分析单客户单次 case）。
- 不对接 TikTok Ads API / MMM 平台 API（V2 再做）。

### 1.4 北极星指标

| 阶段 | 北极星指标 | V1 目标值 |
|------|-----------|----------|
| 内部采用 | PM/AM 月度主动使用次数 | ≥ 20 个 case/月 |
| 分析质量 | 工具识别出的"最佳区间"在客户后续投放中能复现的比例 | ≥ 60% |
| 效率 | 单个 case 从数据上传到输出结论耗时 | < 10 分钟 |

## 2. 用户与场景

### 2.1 核心用户

V1 面向 **TikTok Brand Ads PM 团队自己使用**。典型使用者：

- **产品经理（主用户）**：做 case 复盘、沉淀方法论、撰写 insight 报告。
- **Ad Measurement Science 同事**（次要）：验证指标假设、给 PM 提供口径审核。

V2+ 再扩展到销售/AM 面向客户场景（超出本 PRD 范围）。

### 2.2 核心使用场景

**场景 A：客户 QBR 复盘**
PM 拿到客户过去 6 个月每周的投放数据 + MMM 周 ROI 输出，想回答："这个客户的 MMM ROI 主要受哪个投放指标驱动？最佳区间是多少？"

**场景 B：跨 campaign 对比**
PM 有 10 个 campaign 的聚合指标和 MMM campaign-level ROI，想找出"什么样的投放特征 ROI 最高"。

**场景 C：方法论沉淀**
PM 想快速跑几个客户的数据，提炼行业共性 learning，形成 best practice 文档。

### 2.3 用户故事

- 作为 PM，我希望上传一份 Excel/CSV，10 分钟内看到指标相关性排序，这样我就能在 QBR 前准备好数据故事。
- 作为 PM，我希望工具告诉我 frequency 在 3–5 时 ROI 最好，这样我可以给客户一个具体的优化建议。
- 作为 PM，我希望看到每个相关性的置信度/显著性，这样我不会因为数据点太少而误导客户。

### 3.2 核心用户流程

```
[1] 上传数据
    ↓
[2] 字段映射（告诉工具哪列是 CTR，哪列是 MMM ROI）
    ↓
[3] 数据预览与清洗（剔除异常周/异常 campaign）
    ↓
[4] 选择分析类型（相关性 / 最佳区间 / 两者都要）
    ↓
[5] 查看结果
    ├── 相关性排序 + 热力图
    ├── 每个指标的最佳区间图（分箱柱状图 + ROI 均值）
    └── 多指标联合重要性（SHAP）
    ↓
[6] 导出 Insight 卡片（Markdown / PNG / 可粘贴到飞书）
```

### 3.3 页面与交互要点

#### 页面 1：数据上传页
- 接受 .csv / .xlsx
- 显示示例模板下载链接（含标准列名）
- 上传后预览前 20 行，自动识别列类型（数值/日期/文本）

#### 页面 2：字段映射页
- 左侧：用户文件的列名
- 右侧：系统标准字段（下拉选）
- **必填字段**：时间周期、MMM ROI
- **可选指标字段**：CTR、VTR、CPM、CPM 波动系数、Frequency、Reach、Impressions、Spend、6s+ View Rate、Completion Rate
- 支持"自定义指标"（用户可添加不在预设列表中的列）

#### 页面 3：分析结果页

**Tab A：相关性分析**
- 横向条形图：每个指标与 MMM ROI 的相关系数（正/负着色）
- 每条附带：相关系数值、p-value、样本量
- 右侧热力图：所有指标之间的两两相关性（识别多重共线性）

**Tab B：最佳区间**
- 选择单个指标后：
  - 分箱柱状图：横轴为该指标的分箱区间（默认 5 箱，可调），纵轴为该箱内的 MMM ROI 均值
  - 高亮 ROI 最高的箱，标注区间
  - 附带：该区间的样本数、ROI 均值 & 95% 置信区间
- 决策树可视化（深度 ≤ 3）：展示多指标如何组合划分高/低 ROI 群

**Tab C：Insight 卡片**
- 自动生成文字版结论，例如：
  > 在 2025-Q4 客户 X 的数据中：
  > - Frequency 与 MMM ROI 相关性最强（Spearman ρ = 0.62, p < 0.01, n=52）
  > - Frequency 在 3.5–5.2 区间时 ROI 均值最高（箱内均值 2.8x，其他箱 1.6–2.1x）
  > - CTR 相关性弱（ρ = 0.12, p = 0.38），不建议作为优化杠杆
- 支持一键复制为 Markdown / 导出 PNG

### 3.4 交互细节

- 所有图表支持 hover 查看数据点详情
- 所有分析参数（分箱数、相关性方法、置信度水平）都可调，默认值来自下方 §4 方法论
- 不对样本量设硬性阻断。工具会根据样本量自动切换推荐的分析方法（详见 §4.3），并在结果顶部标注"小样本场景"提示和适配后的方法说明，但始终允许运行

## 4. 算法 / 分析方法论

### 4.1 相关性分析

| 方法 | 使用条件 | V1 是否默认启用 |
|------|---------|----------------|
| Pearson | 两变量近似线性、正态 | ✅ 默认展示 |
| Spearman（秩相关） | 单调但非线性、抗异常值 | ✅ 默认展示（推荐） |
| Kendall τ | 样本量小（n<30）时更稳健 | 🔸 可选 |
| MIC / 互信息 | 捕捉非单调关系 | V2 |

**默认策略**：同时计算 Pearson 和 Spearman，若两者差异 > 0.2 则在界面上标黄提示"可能存在非线性关系"。

**显著性**：所有相关性输出 p-value，p < 0.05 标绿、0.05–0.1 标黄、> 0.1 标灰。

**多重检验校正**：当同时比较 ≥ 5 个指标时，对 p-value 采用 Benjamini-Hochberg FDR 校正，避免假阳性。

### 4.2 最佳区间识别

**Step 1：单指标分箱（Binning）**

- 默认等频分箱（quantile binning），每箱样本量相等，避免尾部空箱
- 默认箱数 = 5，用户可调（3–10）
- 每箱输出：箱区间、样本数、MMM ROI 均值、95% 置信区间（bootstrap 或 t 分布）
- **最佳区间 = MMM ROI 均值最高且与最低箱差异显著（t-test p<0.05）的箱**

**Step 2：多指标联合识别（决策树）**

- 用 CART 决策树，depth ≤ 3，min_samples_leaf = max(5, n/10)
- 目标变量：MMM ROI
- 输出：叶节点的平均 ROI + 对应的多维区间规则，例如：
  > "当 Frequency ∈ [3.5, 5.2] 且 VTR > 45%，MMM ROI 均值 = 3.1x"

**Step 3：特征重要性（V1 基础版 + V2 SHAP）**

- V1：用梯度提升树（XGBoost/LightGBM）的 feature_importance 输出各指标的相对重要性
- V2：输出 SHAP 值，支持每个样本的单点解释

### 4.3 小样本适配策略（不设样本量硬阻断）

现实上，品牌客户单次 case 经常只有 4–5 个周期的 MMM 输出。工具**不设样本量下限**，任何 n ≥ 3 的数据都允许运行。工具根据样本量自动切换为更合适的方法，并在界面上透明披露选择和局限。

| 样本量 n | 默认相关性方法 | 最佳区间方法 | 结果展示策略 |
|----------|--------------|-------------|-------------|
| n < 6 | Kendall τ（最抗小样本）+ 简单排序 | 不做分箱，直接按单样本逐点排序展示 ROI vs. 指标散点 | 不展示 p-value（无统计意义），改为展示"趋势方向"+"数据点列表"；Insight 卡片措辞降级为"观察到的模式" |
| 6 ≤ n < 10 | Kendall τ + Spearman 双展示 | 最多 2–3 箱的粗分箱 | 展示 p-value 但标注"低功效"；置信区间用 bootstrap（1000 次重抽样） |
| 10 ≤ n < 20 | Spearman 为主 | 默认 3 箱 | 标注"小样本场景"；置信区间用 bootstrap |
| 20 ≤ n < 50 | Spearman + Pearson | 默认 5 箱 | 标准展示，提示"推荐 Spearman" |
| n ≥ 50 | 全方法启用 | 默认 5 箱，支持决策树 | 全部功能正常 |

**小样本下的关键设计**：

1. **方法自适应而非拒绝运行**：永不阻断用户。n=4 也可以跑，只是输出形态不同。
2. **语言降级**：n<6 时 Insight 卡片不使用"显著相关""最佳区间"等强结论词，改为"观察到 X 与 ROI 同向/反向变动"这类描述性表达。
3. **强制 disclaimer**：n<10 时 Insight 卡片顶部自动插入"小样本预警"提示，明确指出结论置信度低，建议结合业务判断。
4. **数据点全披露**：小样本下直接在界面展示所有原始观测点的表格，让 PM 可以肉眼复核——这是数据点少时最实用的做法。
5. **鼓励积累**：V2 加入跨 case pooling 和 Bayesian 先验（用历史客户数据做先验分布），用方法论提升小样本下的可信度。

**时间序列自相关处理**：若数据是时间序列（按周/天），V1 在文档中提示用户"相关性会受自相关影响"，V2 增加差分/去趋势选项。

### 4.4 因果性免责声明（重要）

工具明确标注：**相关 ≠ 因果**。MMM ROI 与投放指标的相关性可能来自共同的外部驱动（季节、预算规模、竞品活动）。工具输出仅作为**假设发现**，不得直接作为客户优化承诺。此 disclaimer 需出现在 Insight 卡片顶部。

## 5. 数据模型与字段定义

### 5.1 输入数据 Schema

用户上传的文件应是**宽表**，一行 = 一个观测单位（周 / 日 / campaign / 地域）。

#### 必填字段

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| `period_id` | string | 观测单位 ID | "2025-W42" / "campaign_abc" |
| `mmm_roi` | float | 该周期客户 MMM 输出的 TikTok 增量 ROI | 2.45 |

#### 推荐指标字段（任选其一或多个）

| 字段名 | 类型 | 单位 | 定义 | 备注 |
|--------|------|------|------|------|
| `ctr` | float | % | Click-Through Rate = 点击数 / 曝光数 × 100 | 0–100 |
| `vtr_6s` | float | % | 6 秒 View-Through Rate = 6s+ 观看数 / 曝光数 | 6s 为品牌默认口径 |
| `vtr_completion` | float | % | 完播率 = 完播数 / 曝光数 | 可选 |
| `cpm` | float | USD | Cost Per Mille | 原币单位需一致 |
| `cpm_volatility` | float | 无单位 | 该周期内 CPM 的变异系数 = std(CPM_daily) / mean(CPM_daily) | 衡量 CPM 稳定度 |
| `avg_frequency` | float | 次 | 平均触达频次 = Impressions / Reach | 品牌广告核心指标 |
| `reach` | int | 人 | 独立触达用户数 | |
| `impressions` | int | 次 | 曝光数 | |
| `spend` | float | USD | 投放花费 | 用于控制变量 |

#### 可选控制变量

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `seasonality_flag` | enum | 节假日/大促/常规 |
| `creative_count` | int | 该周期在投素材数 |
| `placement_mix` | string | 版位组合（TopView / In-Feed / Branded Effect 等） |

### 5.2 字段口径对齐原则

- 所有时间周期必须**等长**（全部按周 或 全部按月），不能混用。
- `mmm_roi` 必须是**增量 ROI**（incremental），而非 last-click ROAS——字段映射页会强制要求用户确认勾选。
- `avg_frequency` 的 reach 口径要与客户 MMM 口径一致（通常是 30 天滚动 reach）。

### 5.3 数据质量校验规则

工具在上传后自动执行：

1. 缺失值：某指标列缺失率 > 30% 时警告并建议剔除该列
2. 异常值：用 IQR × 1.5 方法标记，允许用户一键剔除
3. 方差为 0 的列直接剔除（无分析价值）
4. `mmm_roi` 为负或 > 20：弹确认提示（可能是数据单位错误）
5. 时间周期去重检查

## 7. 开放问题（待讨论）

1. **MMM ROI 的时间滞后**：投放指标在第 T 周，MMM ROI 的 carry-over 效应可能让"真实相关"出现在 T+1 甚至 T+2 周。V1 是否提供"lag 选择器"（如 lag=0/1/2）？建议：V1 只支持同周期 lag=0，在文档中标明此假设，V2 再扩展。
2. **控制变量的引入**：是否在相关性分析时自动 partial out spend 的影响（偏相关系数）？建议：V1 提供"是否控制 spend"开关，默认开启。
3. **跨客户 benchmarking**：当我们积累了 20+ 个客户的 case 后，是否要允许看到"行业中位数最佳 frequency 区间"？涉及数据合规，需法务评审——放到 V2 讨论。
4. **与广告系统的双向集成**：未来是否直接在 TikTok Ads Manager 里嵌入"最佳区间预警"？依赖工程大动作，暂不在本 PRD 范围。

> **Key Finding**
> - **avg_frequency** 与 MMM ROI 相关性最强：Spearman ρ = 0.64 (p = 0.008), n = 13
> - **cpm_volatility** 呈显著负相关：ρ = -0.51 (p = 0.04)
> - **CTR** 相关性弱且不显著：ρ = 0.18 (p = 0.55)
>
> **Optimal Range**
> - avg_frequency ∈ [3.5, 5.0] 区间 MMM ROI 均值 = 2.7x（其他区间 1.4–2.0x）
> - cpm_volatility < 0.15 时 ROI 均值 = 2.5x
>
> **Recommendation Hypothesis**（需客户验证）
> - 维持 frequency 在 3.5–5.0 之间
> - 减少日内预算大幅调整以降低 CPM 波动
>
> ⚠️ **Disclaimer**：以上为相关性分析结果，不代表因果。建议通过 A/B 或增量测试验证。
